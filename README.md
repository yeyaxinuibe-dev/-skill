# 🎙️ interview-review — 面试复盘 Skill for Claude

> 一个面向求职者的 Claude Skill，帮你系统性地复盘面试表现：逐题分析、挖掘意图、生成可下载的结构化复盘报告（.docx）。

---

## ✨ 功能亮点

- **逐题深度分析**：每道面试题输出提问意图 × 亮点 × 不足 × 改进建议 × 参考答案 × 星级评分
- **题型自动识别**：行为类、动机类、能力类、情景类、压力测试、技术类……匹配对应答题框架（STAR / PREP / FAB / SOAR）
- **多格式输入支持**：直接粘贴 / `.txt` / `.md` / `.docx` / `.pdf`，音频文件提示转写工具
- **整体表现总结**：综合评分 + 三大亮点 + 三大待改进项 + 面试官可能印象 + 下一步行动建议
- **一键生成 Word 报告**：含封面、逐题分析、改进计划，命名格式 `面试复盘报告_职位_日期.docx`

## 🚀 快速开始

### 安装方式

将整个 `interview-review/` 文件夹放入 Claude 的 Skills 目录（通常为 `/mnt/skills/user/`），或打包为 `.skill` 文件上传至支持 Skill 的 Claude 环境。

### 触发方式

只需在对话中提到以下任意关键词，Skill 即会自动激活：

> `面试` · `面经` · `复盘` · `面完了` · `刚面了` · `面试题` · `录音` · `面试记录`

不需要显式说"复盘"，Claude 会主动介入。

## 📖 使用示例

**场景一：粘贴面试记录**
```
我刚面完字节的产品岗，面试官问了我三个问题：
1. 说说你做过最有挑战的项目
2. 如果用户投诉某个功能不好用，你怎么处理？
3. 你的职业规划是什么？

我的回答是……（粘贴内容）
```

**场景二：上传文字记录文件**
```
（上传 interview.txt）帮我复盘一下
```

**场景三：口语化描述**
```
我刚面了一家 AI 公司的算法岗，技术轮，聊了大概五道题，感觉发挥一般
```
> Claude 会追问职位方向、轮次、题目内容，再进行分析。

## 📊 评分标准（逐题）

| 评分 | 标准 |
|------|------|
| ★★★★★ 5/5 | STAR 完整 + 有量化结果 + 有反思 + 表达流畅 |
| ★★★★☆ 4/5 | STAR 基本完整，结果略模糊，或表达有小瑕疵 |
| ★★★☆☆ 3/5 | ST 完整，AR 缺失或模糊 |
| ★★☆☆☆ 2/5 | 只有情境，行动和结果都没说清楚 |
| ★☆☆☆☆ 1/5 | 跑题、空洞、无实质内容 |

---

## 🗂️ 题型框架速查

| 题型 | 常见问法 | 推荐框架 |
|------|----------|----------|
| 行为类 | 说说你曾经……/ 举个例子 | STAR（情境-任务-行动-结果） |
| 动机类 | 为什么选我们 / 职业规划 | PREP（观点-原因-例子-重申） |
| 能力类 | 你擅长什么 / 核心竞争力 | FAB（特点-优势-价值） |
| 情景类 | 假设你遇到 / 如果你会怎么做 | SOAR（情境-障碍-行动-结果） |
| 压力测试 | 说你的缺点 / 最大失败 | 真实 + 成长弧（承认-反思-改变） |
| 技术类 | 解释原理 / 你如何实现 | 结构化拆解（定义-原理-应用-边界） |
| 追问结果类 | 效果怎么样 / 提升了多少 | STAR 的 AR 部分 + 量化数据 |

## 🛠️ scripts/parse-transcript.py

上传 `.txt` / `.md` 文件时自动调用，将常见格式的面试记录解析为结构化 JSON：

```bash
python scripts/parse-transcript.py interview.txt
```

支持以下格式：
- `Q: ... A: ...`
- `问：... 答：...`
- `面试官：... 我：...`
- 编号列表（`1. 问题\n回答`）
- 无法识别时兜底返回原文
