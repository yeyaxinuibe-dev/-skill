# 面试复盘 Skill

帮助求职者系统性地复盘面试表现，逐题分析、总结规律、生成可下载的复盘报告。

## 目录结构

```
interview-review/
├── SKILL.md                     ← 本文件，工作流总指挥
├── references/
│   └── format-standards.md     ← 必读：输出格式、评分标准、题型框架、文件命名规范
├── scripts/
│   └── parse-transcript.py     ← 上传文字记录时调用，自动解析为结构化问答
├── assets/                     ← 暂无，Word 报告样式在生成时内联处理
└── evals/
    └── test-cases.json         ← 5个测试用例，覆盖主要使用场景
```

**开始工作前必须先读** `references/format-standards.md`——所有输出格式、评分标准、题型框架都在那里，必须严格遵守。

---

## 工作流

### Step 1：理解输入

| 输入类型 | 处理方式 |
|--------|---------|
| 直接粘贴文字问答 | 直接进入 Step 2 |
| 上传 .txt / .md 文件 | 调用 `scripts/parse-transcript.py <filepath>` 解析为结构化问答 |
| 上传 .docx / .pdf | 用 bash 读取文本内容后进入 Step 2 |
| 上传音频文件 | ⚠️ 无法解析，提示用户先转文字（推荐：剪映 / 讯飞听见 / 飞书妙记） |
| 口语化描述 | 先追问补全关键信息，再进入 Step 2 |

**口语化描述时，先追问：**
- 应聘职位和公司方向（大概即可）
- 面试轮次（HR轮 / 技术轮 / 管理层轮）
- 大概几道题

---

### Step 2：逐题深度分析

严格按照 `references/format-standards.md` 中的「输出格式标准」逐题输出，不得省略任何模块：

```
【题目 N】[题目原文]

🎯 面试官提问意图
📝 你的回答摘要（若无回答则跳过）
✅ 亮点
⚠️ 不足
💡 改进建议 + 参考答案框架
📊 回答质量评分：★ X/5 + 一句话理由
```

即使用户没有提供自己的回答，也必须完成提问意图分析和参考答案输出。

---

### Step 3：整体复盘总结

所有题目分析完成后输出（格式详见 `references/format-standards.md`）：

- 综合评分（1-10分）+ 理由
- 三大亮点 / 三大待改进项
- 面试官可能的印象
- 下一步行动建议（具体可操作）

---

### Step 4：生成 Word 报告（可选）

整体总结后主动询问用户是否需要 Word 报告。确认后：
1. 读取 `/mnt/skills/public/docx/SKILL.md` 获取生成规范
2. 生成含封面、逐题分析、改进计划的 .docx
3. 文件命名遵守 `references/format-standards.md` 命名规范
4. 用 `present_files` 提供下载

---

## 注意事项

- 保持客观、建设性，不过度批评，不空洞鼓励
- 改进建议必须具体可操作，禁止「多练习」等模糊表述
- 涉及特定行业时，调整分析角度和行业术语
