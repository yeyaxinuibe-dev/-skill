# 面试复盘格式规范

## 输出格式标准

### 逐题分析（每题必须包含以下全部模块，不得省略）

```
【题目 N】[题目原文]

🎯 面试官提问意图
📝 你的回答摘要
✅ 亮点
⚠️ 不足
💡 改进建议 + 参考答案框架
📊 回答质量评分：[★☆] X/5  + 一句话理由
```

### 整体总结（每次复盘结束后必须输出）

```
## 整体表现总结
### 综合评分（1-10分）
### 三大亮点
### 三大待改进项
### 面试官可能的印象
### 下一步行动建议
```

---

## 内容规范

### 评分标准（逐题）

| 分数 | 标准 |
|------|------|
| 5/5 | STAR完整 + 有量化结果 + 有反思 + 表达流畅 |
| 4/5 | STAR基本完整，结果略模糊，或表达有小瑕疵 |
| 3/5 | ST完整，AR缺失或模糊 |
| 2/5 | 只有情境，行动和结果都没说清楚 |
| 1/5 | 跑题、空洞、无实质内容 |

### 改进建议规范
- 必须具体可操作，禁止出现「多练习」「多准备」等模糊表述
- 每道题必须给出至少1个完整的参考答案示例（即使用户没有提供自己的回答）
- 参考答案要匹配题型对应框架（见 format-standards.md 题型表）

### 语气规范
- 保持客观、建设性，不过度批评，不空洞鼓励
- 指出不足时先肯定素材价值，再说缺什么
- 禁止使用「你的回答很好」「非常棒」等无依据的夸奖

## 题型识别 & 框架映射

| 题型 | 关键词 | 推荐框架 |
|------|--------|---------|
| 行为类 | 说说你曾经 / 举个例子 / 描述一次 | STAR（情境-任务-行动-结果） |
| 动机类 | 为什么选我们 / 为什么换工作 / 职业规划 | PREP（观点-原因-例子-重申） |
| 能力类 | 你擅长什么 / 最大优点 / 核心竞争力 | FAB（特点-优势-价值） |
| 情景类 | 假设你遇到 / 如果...你会怎么做 | SOAR（情境-障碍-行动-结果） |
| 压力测试 | 说你的缺点 / 最大失败 / 最大困难 | 真实+成长弧（承认+反思+改变） |
| 技术类 | 解释XX原理 / 你如何实现 / 代码/方案题 | 结构化拆解（定义-原理-应用-边界） |
| 追问结果类 | 效果怎么样 / 提升了多少 / 最终结果 | STAR的AR部分 + 量化数据 |

## 文件命名规范

生成的 Word 报告统一命名为：
```
面试复盘报告_[职位简称]_[日期YYYY-MM-DD].docx
```

示例：
- `面试复盘报告_产品经理_2026-03-18.docx`
- `面试复盘报告_AI工程师_2026-03-18.docx`
- `面试复盘报告_未知职位_2026-03-18.docx`（职位不明时使用）
